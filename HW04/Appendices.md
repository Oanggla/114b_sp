## 附錄 A：SIC/XE 指令集與操作碼 (OPTAB) 對照表

下表收錄了 SIC/XE 最常用的指令助憶碼、指令格式、操作碼 (Opcode) 以及受其影響的暫存器與功能說明。

| 助憶碼 (Mnemonic) | 格式 (Format) | 操作碼 (Opcode) | 運算元 | 說明與功能描述 |
| :--- | :---: | :---: | :--- | :--- |
| **`ADD`** | 3/4 | `18` | `m` | $A \leftarrow (A) + (m)$ （將記憶體內容加至暫存器 A） |
| **`ADDR`** | 2 | `90` | `r1, r2` | $r2 \leftarrow (r2) + (r1)$ （暫存器相加） |
| **`AND`** | 3/4 | `40` | `m` | $A \leftarrow (A) \land (m)$ （位元 AND 運算） |
| **`CLEAR`** | 2 | `B4` | `r1` | $r1 \leftarrow 0$ （將暫存器清空為 0） |
| **`COMP`** | 3/4 | `28` | `m` | 比較 $(A)$ 與 $(m)$ 的大小，並設定條件碼 (CC) |
| **`COMPR`** | 2 | `A0` | `r1, r2` | 比較 $(r1)$ 與 $(r2)$ 的大小，並設定條件碼 (CC) |
| **`DIV`** | 3/4 | `24` | `m` | $A \leftarrow (A) / (m)$ （暫存器 A 除以記憶體內容） |
| **`DIVR`** | 2 | `9C` | `r1, r2` | $r2 \leftarrow (r2) / (r1)$ （暫存器相除） |
| **`J`** | 3/4 | `3C` | `m` | $PC \leftarrow m$ （無條件跳躍） |
| **`JEQ`** | 3/4 | `30` | `m` | 若條件碼為「等於」(CC = `=`)，則跳躍至 $m$ |
| **`JGT`** | 3/4 | `34` | `m` | 若條件碼為「大於」(CC = `>`)，則跳躍至 $m$ |
| **`JLT`** | 3/4 | `38` | `m` | 若條件碼為「小於」(CC = `<`)，則跳躍至 $m$ |
| **`JSUB`** | 3/4 | `48` | `m` | $L \leftarrow (PC)$, $PC \leftarrow m$ （呼叫子程式，保存返回位址） |
| **`LDA`** | 3/4 | `00` | `m` | $A \leftarrow (m)$ （載入記憶體內容至暫存器 A） |
| **`LDB`** | 3/4 | `68` | `m` | $B \leftarrow (m)$ （載入記憶體內容至基底暫存器 B） |
| **`LDCH`** | 3/4 | `50` | `m` | $A[\text{rightmost byte}] \leftarrow (m)$ （載入單一字元至暫存器 A） |
| **`LDT`** | 3/4 | `74` | `m` | $T \leftarrow (m)$ （載入記憶體內容至暫存器 T） |
| **`MUL`** | 3/4 | `20` | `m` | $A \leftarrow (A) \times (m)$ （暫存器 A 乘以記憶體內容） |
| **`RD`** | 3/4 | `D8` | `m` | 從裝置 $m$ 讀取一個位元組資料至暫存器 A |
| **`RSUB`** | 1/3/4 | `4C` | 無 | $PC \leftarrow (L)$ （自子程式返回） |
| **`STA`** | 3/4 | `0C` | `m` | $m \leftarrow (A)$ （將暫存器 A 的內容儲存至記憶體） |
| **`STB`** | 3/4 | `78` | `m` | $m \leftarrow (B)$ （將暫存器 B 的內容儲存至記憶體） |
| **`STCH`** | 3/4 | `54` | `m` | $m \leftarrow (A)[\text{rightmost byte}]$ （將 A 暫存器之單一字元寫入記憶體） |
| **`SUB`** | 3/4 | `1C` | `m` | $A \leftarrow (A) - (m)$ （暫存器 A 減去記憶體內容） |
| **`SUBR`** | 2 | `94` | `r1, r2` | $r2 \leftarrow (r2) - (r1)$ （暫存器相減） |

## 附錄 B：SIC/XE 定址模式與計算公式速查

在 SIC/XE 中，目標位址 $TA$（Target Address）與實際提取的運算元（Operand）會根據機器碼中的 **n, i, x, b, p, e** 六個旗標位元而有所不同。

### B.1 運算元取得模式 (n, i)
這兩個位元決定了 $TA$ 計算完成後，程式如何解讀該位址的內容：

1. **立即定址 (Immediate Addressing)**：`n = 0, i = 1`
   - 運算元為 $TA$ 的**數值本身**。不進行任何記憶體存取。
2. **間接定址 (Indirect Addressing)**：`n = 1, i = 0`
   - 程式先讀取記憶體位址 $TA$ 處的數值，該數值被視為一個指標，指向真正的運算元位址。
   - 運算元 = $\text{Memory}[\text{Memory}[TA]]$
3. **直接定址 (Simple/Direct Addressing)**：`n = 1, i = 1`（或 `n = 0, i = 0` 相容模式）
   - 記憶體位址 $TA$ 處所儲存的數值即為運算元。
   - 運算元 = $\text{Memory}[TA]$

### B.2 目標位址 $TA$ 計算公式 (x, b, p, e)
目標位址的計算取決於位移量 `disp` (12 bits) 或絕對地址 `address` (20 bits)：

| 定址類型 | 旗標狀態 (b, p, e) | 運算式 (若 x = 0) | 運算式 (若 x = 1) |
| :--- | :---: | :--- | :--- |
| **絕對定址 (Format 3)** | `b = 0, p = 0, e = 0` | $TA = \text{disp}$ | $TA = \text{disp} + (X)$ |
| **絕對定址 (Format 4)** | `b = 0, p = 0, e = 1` | $TA = \text{address}$ | $TA = \text{address} + (X)$ |
| **PC 相對定址** | `b = 0, p = 1, e = 0` | $TA = (PC) + \text{disp}$ | $TA = (PC) + \text{disp} + (X)$ |
| **Base 相對定址** | `b = 1, p = 0, e = 0` | $TA = (B) + \text{disp}$ | $TA = (B) + \text{disp} + (X)$ |

- **PC 相對定址注意事項**：位移量 `disp` 為 12 位元**有號二補數**，表達範圍為 $-2048$ 至 $+2047$。
- **Base 相對定址注意事項**：位移量 `disp` 為 12 位元**無號數**，表達範圍為 $0$ 至 $4095$。

## 附錄 C：學生手動實作專案指引 (Hands-on Lab Projects)

為了加深對底層系統程式運作的理解，建議學生動手實作以下三個經典專案。這些專案彼此關聯，若全部完成，便能在電腦中模擬出一個簡易的系統開發環境。

### 專案一：以 Python/C++ 實作一個 SIC/XE 虛擬機器 (Emulator)
* **專案目標**：寫一個程式，以軟體方式模擬一台擁有 CPU 暫存器、1 MB 記憶體空間的虛擬電腦，並能夠正確執行 SIC/XE 的機器指令。
* **核心實作內容**：
  1. **記憶體與暫存器定義**：使用陣列模擬 1 MB 的 RAM，並宣告變數模擬 A, X, L, B, S, T, PC, SW 等暫存器。
  2. **載入器功能**：讀取組譯器產生的目的檔（`.obj` 檔案中的 H、T、M、E 紀錄），將機器碼寫入模擬記憶體的對應位址。
  3. **Fetch-Decode-Execute 循環**：
     - 從記憶體中以 `PC` 位址讀取指令位元組。
     - 解析指令格式（Format 1, 2, 3, 4）與旗標旗標（n, i, x, b, p, e），正確計算 $TA$。
     - 根據運算碼（Opcode）執行對應的運算（如算術運算、暫存器存取、分支跳躍）。
  4. **簡易 I/O**：以標準輸入和輸出模擬 SIC/XE 中特定的輸入與輸出裝置。

### 專案二：實作一個支援完整定址模式的二行程組譯器 (Two-Pass Assembler)
* **專案目標**：撰寫一個組譯器，將 SIC/XE 組合語言程式碼（`.asm`）翻譯為可由模擬器執行的機器碼（`.obj`）。
* **核心實作內容**：
  1. **Pass 1 階段**：
     - 解析每一行程式碼的 Label、Opcode 與 Operand。
     - 維護位置計數器 `LOCCTR`，處理記憶體保留虛擬指令（`RESB`, `RESW` 等）。
     - 建立符號表 `SYMTAB`。若有標記重複定義，需輸出語法錯誤。
  2. **Pass 2 階段**：
     - 根據運算碼與 `SYMTAB` 計算定址。
     - 自動判定並嘗試使用 **PC 相對定址**；若超出範圍，嘗試使用 **Base 相對定址**；若皆無法定址，提示錯誤或切換為 Format 4。
     - 產生正確的二進位機器指令。
  3. **目的檔生成**：輸出標準的 ASCII 格式 `.obj` 檔案，包含正確的 H、T、M、E 紀錄，尤其是 Format 4 絕對定址所需的 M 紀錄。

### 專案三：實作一個簡易的 One-Pass 巨集處理器 (Macro Processor)
* **專案目標**：實作一個獨立的巨集處理器，作為組譯器的預處理器（Preprocessor）。它能讀入包含巨集定義與呼叫的源碼，輸出完全展開後的組合語言程式碼。
* **核心實作內容**：
  1. **資料結構建立**：設計 `DEFTAB`、`NAMTAB` 與 `ARGTAB`。
  2. **One-Pass 文字替換**：
     - 當讀到 `MACRO` 時，進入定義模式，讀取參數並將其改寫為位置符號（如 `?1`, `?2`），將指令寫入 `DEFTAB`。
     - 當在源碼中偵測到巨集呼叫，將實際參數填入 `ARGTAB`，切換到展開模式（`EXPANDING = TRUE`），從 `DEFTAB` 讀取並替換輸出，直至 `MEND`。
  3. **進階功能（加分項）**：
     - 實作「唯一標記自動生成（以 `$` 開頭的 label）」，使同一個巨集呼叫多次時，不會產生 label 重複問題。
     - 支援關鍵字參數與預設值。
